Présentation

Le HTML, c'est quoi et pourquoi faire?

HTML pour HyperText Markup Language est le langage standard pour créer des pages web. En HTML vous pourrez structurez vos données de manière sémantique , les organiser de façon logique et même créer des documents interactifs. Vous aurez la possiblilité d'intégrer directement des éléments multimédias. Il a été inventé pour créer des documents statiques qui peuvent se lier façilement aux débuts. Une fois les bases du HTML aquises vous pourrez découvrir d'autres languages pour multiplier vos possibilités de créations et réaliser des documents riches, interactifs, des jeux, des graphiques dynamiques, des applications connectés et bien plus encore.

Evolutions

Le HTML est défini par le World Wide Web Consortium. En ce moment même le HTML est toujours en cours de définition. Le w3c est la réfèrence ultime pour le HTML. Le w3c s'occupe de la création des spécification, et de la recommandation des bonnes pratiques. Ensuite les développeurs de navigateurs web peuvent implémenter les idées du w3c pour les rendres utilisables. Ces dernières annés le HTML a beaucoup changé notamment grâce à l'utilisation grandissante des smartphones pour visiter le web

Et comment je fais pour créer du HTML moi ?

En suivant ce guide vous allez tout savoir pour bien commencer. Le HTML est un language intermediare entre l'ordinateur et les sens de la créature qui s'y intéresse. Je veux simplement dire que le html est purement textuel et que le reste est genéré par un programme appelé navigateur web. Pour que le navigateur le fasse rapidement et simplement le html est un language avec des règles précises. C'est la syntaxe. Ce premier tutoriel va donc surtout se consacrer à expliquer comment écire du HTML juste.

Introduction à l'écriture de HTML

Ouvrez un simple éditeur de texte, c'est là que vous allez écrire le code HTML. Si vous avez WebSpree d'ouvert, c'est parfait car il contient un éditeur de texte intégré. Il faut commencer par indiquer que c'est un document HTML en écrivant <!DOCTYPE html>. Ensuite il faut mettre tout le reste entre <html> et </html>. Et de la même manière il faut toujours mettre votre contenu entre la balise ouvrante et fermante qui correspond à la sémantique du contenu. La balise ouvrante se différencie <nom de la balise> de la balise fermante </nom de la balise> par un slash ( / ) en moins. Il existe aussi des balises autofermantes qui n'ont pas de contenu qui s'écrivent <nom>. Tout cela peut sembler compliqué. La logique est très simple: Tout ce que vous écrivez entre une balise ouvrante et une balise fermante est dedans tout le reste est en dehors et les balises autofermantes n'ont pas de sens avec quelque chose dedans. Les balises peuvent s'emboiter comme des poupées russes. Un élément X est la balise ouvrante X + son contenu + la balise X fermante.

Différents types d'élements

Ici nous avons vu html et p. Tout le code doit se trouver dans l'élément html. L'élément p est là pour 1 paragraphe de texte. Un paragraphe par élément p, pas plus pas moins. Le doctype et la validité

Le doctype est nécessaire en début de fichier. <!doctype html>

Pour qu'un document soit valide il faut que toute balise ouverte retrouve une balise fermante, et les balises n'ont pas le droit de s'entre croiser. Exemple d'interdit:

<p> <strong> bla bla </p> </strong>

On ne peut pas dire quoi est dans quoi.Voici la version juste, où on voit de suite que strong est dans p :

<p> <strong> bla bla </strong> </p>
